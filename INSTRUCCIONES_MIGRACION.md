# 🚀 Guía de Migración de Ecosistema Antigravity

Este documento es tu "Llave Maestra" para replicar la inteligencia, el diseño y los estándares de este workspace en cualquier nuevo proyecto (Control de Gastos, Dashboards, Automatizaciones, etc.).

---

## ⚡ 1. Lo que YA está instalado (Global)
Las siguientes **Skills** viven en tu sistema (`~/.gemini/antigravity/skills/`) y estarán disponibles automáticamente en cualquier carpeta que abras con Antigravity:
- `analizador-proyectos`: Para planificar nuevas arquitecturas.
- `knowledge-manager`: Para leer tus reglas de negocio.
- `workflow-executor`: Para ejecutar procesos paso a paso.

---

## 📂 2. Lo que DEBES COPIAR (Local)
Para que un nuevo proyecto "herede" tu ADN de desarrollo, copia estas carpetas a la raíz de tu nuevo proyecto:

1. **`Global_Rules/`**: Asegura que yo mantenga la mentalidad Senior, SOLID y segura.
2. **`knowledge/`**: Para que yo recuerde tus estilos de diseño y estrategias de marketing.
3. **`.agent/workflows/`**: Para que tus procesos (como `inicio_proyecto.md`) estén listos para ser ejecutados.

---

## 🛠 3. Proceso para un Nuevo Proyecto (Ejemplo)

Si hoy inicias un **"Sistema de Control de Gastos"**, sigue estos pasos:

### Paso 1: Inicialización
Crea la carpeta y copia las carpetas del punto anterior. Luego, abre Antigravity y di:
> *"Analiza mis Global_Rules y mi Knowledge Base. Estamos iniciando un nuevo proyecto de Control de Gastos."*

### Paso 2: Adquisición de Skills Específicas (Opcional)
Antes de empezar a codificar, pregúntate si hay algún skill experto en la comunidad que nos pueda servir. Usa el **skill-matcher** de la siguiente forma:
> *"Busca en el repositorio de skills si hay alguna herramienta para diseño de base de datos o APIs que podamos usar en este Control de Gastos. Evalúa y recomienda."*

### Paso 3: Análisis de Requerimientos
Usa la habilidad de análisis para que yo diseñe la estructura:
> *"Usa la skill **analizador-proyectos** para diseñar la arquitectura de este Control de Gastos. Prioriza el diseño estilo Fintech Premium que tenemos en knowledge."*

### Paso 4: Ejecución de Procesos
Para montar el entorno rápido:
> *"Ejecuta workflow **inicio_proyecto**"*.

---

## 🧠 4. Evolución del Conocimiento
Cada vez que aprendamos algo nuevo o definamos una regla en un proyecto, recuerda:
- **¿Es una regla de código?** Guárdala en `1_Estandares_Core`.
- **¿Es un estilo visual o estrategia?** Crea un nuevo archivo en `knowledge/`.
- **¿Es un proceso repetitivo?** Crea un `.md` en `.agent/workflows/`.

---
**Con este flujo, cada proyecto será mejor y más rápido que el anterior. ¡A construir!**
